1. OpenSBI 是什么？
- What：OpenSBI 是 RISC-V 架构的 Supervisor Binary Interface 的一个参考实现。也是一个固件
- 它运行在机器模式（M Mode）：RISC-V 有多个特权级（M、S、U）。OpenSBI 运行在最高特权级 M Mode，拥有对硬件的完全控制权。
- 它向下管理硬件：负责初始化硬件（如 PLIC 中断控制器、UART 串口等），并处理一些只有 M Mode 才能做的操作（比如关中断、时钟设置）。
- 它向上提供服务：为运行在 S Mode（监管者模式）的操作系统内核提供一组标准调用（SBI 调用）。例如，内核想打印字符、获取时间、关闭系统，就会触发 ecall 指令，陷入 OpenSBI 去执行。
- **SBI 版本注意**：OpenSBI v1.3 运行时 SBI 版本为 1.0，已移除旧的 Console 扩展（EID=0x4）。内核必须使用 Legacy Console 扩展（EID=0x1, FID=0x0）才能正常输出字符。

2. 固件
- 固件在ROM和Flash中
- 功能：初始化硬件、加载真正的操作系统

3. QEMU virt 平台
QEMU 模拟了一个从 0x1000 开始的"ROM"区域。
你通过 -bios 参数指定 OpenSBI 的二进制文件（如 opensbi-riscv64-generic-fw_dynamic.bin），QEMU 会把它加载到该虚拟 ROM 地址。

4. 引导流程
复位 → 0x1000 (OpenSBI 固件) → 初始化 → 跳转到 0x80000000 (内核)

```
硬件复位
    ↓
┌─────────────────────────────┐
│     OpenSBI (固件)          │  ← QEMU 默认加载
│     地址: 0x80000000        │
│                             │
│  - 初始化硬件               │
│  - 设置时钟、中断           │
│  - 提供 SBI 接口           │
│    (Legacy Console EID=0x1) │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│     加载内核 (kernel.elf)   │  ← make run 时 -kernel 参数
│     地址: 0x80200000        │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  跳转执行 _start (start.S)  │
│                             │
│  1. 设置 sp = _stack_top    │
│  2. jal main                │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│     main.c                  │
│     - sbi_console_putchar() │
│     - 输出 "Hello Kernel!"  │
└─────────────────────────────┘
```

5. SBI 调用约定（SBI v0.2+）
- `a0-a5`：参数
- `a6`：Function ID (FID)
- `a7`：Extension ID (EID)
- `ecall`：触发调用

**常见错误**：
- 使用 EID=0x4（Console 扩展）会导致调用失败，因为 SBI v1.0 已移除此扩展
- 正确做法是使用 EID=0x1（Legacy Console）的 FID=0x0（putchar）和 FID=0x1（getchar）
